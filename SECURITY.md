## 报告安全漏洞

请通过以下方式报告安全漏洞：

- **GitHub Issues**: [提交问题](https://github.com/xiaomutou-git/KnowledgeDetective/issues)
- **电子邮件**: 2581687833@qq.com

### 漏洞报告应包含

- 受影响的版本
- 漏洞描述
- 复现步骤
- 潜在影响
- 建议的修复方案（如有）

## 漏洞响应流程

1. 收到漏洞报告后，我们会在 24 小时内确认收到
2. 在 72 小时内评估漏洞的严重性和影响范围
3. 制定修复计划并在 1 周内发布安全更新
4. 在 CHANGELOG.md 中记录安全修复

## 安全最佳实践

### API Key 安全

- 不要将 API Key 硬编码在代码中
- 使用环境变量配置 API Key
- 不要提交包含真实 API Key 的配置文件到版本控制

### 输入验证

- 对所有用户输入进行验证和过滤
- 限制请求体大小
- 防止路径遍历攻击

### 依赖安全

- 定期更新依赖
- 使用 `npm audit` 检查依赖漏洞
- 只使用可信的第三方库
